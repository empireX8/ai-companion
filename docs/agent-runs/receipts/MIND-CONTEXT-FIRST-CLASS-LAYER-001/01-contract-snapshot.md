# Contract Snapshot

Relevant product truth:
- MindLab is an evidence-backed personal understanding engine.
- Mind Context / Context Profile is stable but correctable context about the user.
- Mind Context must be inspectable and correctable.
- If Orvek uses context invisibly, it risks hidden profiling.
- User correction is first-class evidence.
- Facts require receipts.
- Orvek proposes models; it does not issue verdicts.
- No hidden model movement.

Implications for this phase:
- Context items must be selectable objects, not just read-only labels.
- The user must be able to inspect the current read, supporting evidence shape, uncertainty, and linked evidence paths.
- The correction path must create evidence input, not claim the model has already changed.
