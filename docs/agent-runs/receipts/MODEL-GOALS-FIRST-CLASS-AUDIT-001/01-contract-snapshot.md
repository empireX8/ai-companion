# Contract Snapshot

Model Goals are durable objects for goals, anti-goals, standards, priorities, constraints, build direction, and recurring behaviors the user wants to build or stop repeating.

Required contract shape
- First-class and inspectable.
- Evidence-linked.
- User-correctable.
- Tied to model movement and re-entry.
- Distinct from generic productivity goals, generic memories, and generic references.

Behavioral rules
- User correction is first-class evidence.
- Orvek proposes models; it does not issue verdicts.
- No hidden model movement.
- Claims about the user must trace back to evidence or remain clearly provisional.

Repair direction implied by the contract
- If Model Goals are surfaced, they should be presented as a model layer with explicit correction and evidence lineage.
- Any correction path should make clear that correction creates evidence or review input, not an immediate model rewrite unless one has actually occurred.
