# Contract Snapshot

## 2A — Target User + Mental State

- Target user: people stuck in repeated internal loops who need an evidence-backed understanding system, not a journal, chatbot, coach, or mood tracker.
- Mental state: chronic frustration, low bandwidth during the loop, and a need for proof of change rather than reassurance.

## 2B — Trust Mechanics

- Facts require receipts.
- Inferences require supporting receipts and confidence rationale.
- Uncertainty must be labeled and tied to missing evidence.
- Challenge must point to the evidence state, not the user’s identity.
- Orvek proposes models; it does not issue verdicts.
- User correction is first-class evidence.
- Hidden model movement is not allowed.
- Relief is allowed only as a byproduct of clarity.
- Fieldwork must generate evidence, test a claim, clarify uncertainty, or move a decision.
- Weak evidence must produce weak claims.

## 2C — Core Nouns

- Life Data, Evidence, Receipt, Object, Claim Object, Model Goal, Mind Context / Context Profile, Model Movement / What Changed, Reality Gate, Fieldwork, Watch For, Active Question, Re-entry.
- Important guardrail: do not ban individual words everywhere; flag only when the term creates the wrong Orvek meaning or violates the trust contract.

## 2D — Output Grammar

- Formal Report / What Changed must expose:
  - Evidence
  - Facts
  - Movement
  - Supported read
  - Disconfirming evidence
  - Uncertainty
  - Impact
  - Reality Gate
  - Fieldwork / Watch For
  - Re-entry
  - What would change this conclusion
- Today / Re-entry must expose:
  - State
  - Delta
  - Why it matters
  - Evidence pointer
  - Next observation / test
  - Re-entry trigger
  - Capture path
