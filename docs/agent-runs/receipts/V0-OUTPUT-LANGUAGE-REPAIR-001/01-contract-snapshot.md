# Contract Snapshot

## 2A - Target user and state

- Target user: someone losing time and confidence to unresolved internal loops.
- Mental state: low bandwidth, chronic frustration, and a need for proof of change.

## 2B - Trust mechanics

- Facts require receipts.
- Inferences require supporting receipts and confidence rationale.
- Uncertainty must be labeled and tied to missing evidence.
- Challenge must point to the evidence state, not the user.
- Orvek proposes models; it does not issue verdicts.
- User correction is first-class evidence.
- Hidden model movement is not allowed.
- Relief is allowed only as a byproduct of clarity.
- Fieldwork must generate evidence, test a claim, clarify uncertainty, or move a decision.
- Weak evidence must produce weak claims.

## 2C - Core nouns

- Life Data, Evidence, Receipt, Object, Claim Object, Model Goal, Mind Context / Context Profile, Model Movement / What Changed, Reality Gate, Fieldwork, Watch For, Active Question, Re-entry.

## 2D - Output grammar

- Report / What Changed:
  - Evidence
  - Facts
  - Movement
  - Supported read
  - Disconfirming evidence
  - Uncertainty
  - Impact
  - Reality Gate
  - Fieldwork / Watch For
  - Re-entry
  - What would change this conclusion
- Today / Re-entry:
  - State
  - Delta
  - Why it matters
  - Evidence pointer
  - Next observation / test
  - Re-entry trigger
  - Capture path
